      Layer-by-layer strategy & economics

      Each tier uses the platform product differently, with distinct economics and P&L levers.

      Bronze
      Enablement

      Controlled self‑service for small operational tweaks with strict guardrails. Goal: scale adoption without creating a review bottleneck while protecting larger systems.

      How it uses GaC/CaC
      • Use only certified nodes + templates; auto‑evidence and telemetry are on by default
      • Sandbox environment; promotion triggers evidence + routing rules
      Economic model
      • Retainer/subscription: platform access + enablement + sandbox (€250k illustrative per division)
      • Consumption budgeting: premium connectors + AI tokens gated behind value
      • Predictable chargeback: replaces ticket-by-ticket billing
      Primary P&L dial
      Retainer/subscription amount (per division) + governance strictness / routing (what is auto‑approved vs. escalated) + token budgets.
      Silver
      Factory

      Smaller custom components and projects. Business leads; T-Tech provides expertise, compliant building blocks, and factory capacity.

      How it uses GaC/CaC
      • Client-side builders assemble solutions using golden paths and standard logging/evidence
      • T-Tech delivers custom compliant components (special nodes/connectors) when needed
      • Reusable outputs graduate into new certified nodes (catalog grows)
      Hybrid‑shore delivery
      • Offshore: build/test, refactoring, documentation, standard workflows
      • Onshore: architecture, stakeholder management, risk/security alignment
      • Quality gates: Golden Paths + review + automated compliance evidence
      Economic model
      • Capacity blocks (retainer‑like) for predictable expert support + delivery bandwidth
      • Small projects: custom components and smaller project pricing (T&M / fixed “small package”)
      • Packaged components: priced add-ons for specialized compliant nodes (reusable accelerators)
      Primary P&L dial
      Project/component pricing + Labour Location Arbitrage (onshore/offshore ratio) + standardization (reuse rate).
      Gold
      SWAT

      High‑impact “big fish” programs (cross‑divisional / high‑risk). Goal: measurable ROI + internal margin.

      How it uses GaC/CaC
      • Use the catalog as a baseline; add premium controls (security sign‑off, pen test, SoD)
      • Harvest generic pieces into new certified building blocks (factory gets stronger)
      What makes something “Gold”
      • High impact: cross‑divisional or mission‑critical outcomes
      • High compliance: security sign‑off + audit evidence is non‑negotiable
      • Measured as ROI/risk: not tickets closed
      Economic model
      • Project pricing: cost‑plus / fixed‑price with SLA add‑on
      • Margin capture: pay for premium governance + delivery certainty
      Primary P&L dial
      Project volume (projects/year) + margin per project (pricing) + catalog lift capture (reusable gains harvested).

      Technology strategy: Co‑Existence Doctrine

      OutSystems + Power Platform

      Protect existing heavy apps on OutSystems; make Power Platform the default for new internal automation.

      Rule 1 — Cap

      No new internal apps on OutSystems unless a strict exception is granted.

      • Stops portfolio sprawl
      • Forces reuse via GaC/CaC + Golden Paths
      Rule 2 — Grow

      All new internal automations default to Power Platform (flows, apps, approvals).

      • Leverages existing M365 investment
      • Governed speed via GaC/CaC (compliance by design)
      Rule 3 — Migrate

      Systematically migrate "Silver" class apps that are over‑engineered on OutSystems.

      • Reduces licensing/maintenance cost
      • Creates reusable components for the GaC/CaC catalog
      When to use what
      • OutSystems: existing "heavy" apps and complex external-facing portals
      • Power Platform: default for internal process automation (Bronze/Silver/Gold by risk)
      Benefits
      • Finance: cost reduction without destabilizing production
      • Speed: clear default platform + certified building blocks
      • Governance: less exception handling, more "compliance by design"
